# 牙齿疾病检测系统 - 部署文档

## 1. 概述

本文档描述了牙齿疾病检测系统的完整部署流程，包括环境准备、服务启动和验证等步骤。

### 1.1 部署架构

```
┌─────────────────────────────────────────┐
│            Nginx (端口 80)              │
└─────────────────────────────────────────┘
                    │
┌─────────────────────────────────────────┐
│         uWSGI (端口 8001)               │
│           └─ Flask 应用                  │
└─────────────────────────────────────────┘
                    │
    ┌───────────────┼───────────────┐
    ▼               ▼               ▼
┌─────────┐   ┌─────────┐   ┌──────────┐
│  Redis  │   │ Celery  │   │  YOLOv5  │
│(6379)   │   │  Worker │   │  + MobileNetV3
└─────────┘   └─────────┘   └──────────┘
```

### 1.2 组件说明

| 组件 | 端口 | 说明 |
|------|------|------|
| Nginx | 80 | Web 服务器（可选，用于负载均衡和静态文件服务） |
| uWSGI | 8001 | WSGI 服务器，运行 Flask 应用 |
| Redis | 6379 | 消息队列，用于 Celery 异步任务 |
| Celery | - | 异步任务处理器 |

## 3. 部署步骤

### 3.1 步骤一：启动 Redis

Redis 作为消息队列，用于 Celery 异步任务处理。

```bash
# 进入 Redis 安装目录
cd /usr/local/bin

# 启动 Redis 服务
redis-server /etc/redis.conf
```

**验证 Redis 启动成功**：

```bash
# 测试 Redis 连接
redis-cli ping
# 返回 PONG 表示成功
```

### 3.2 步骤二：激活 Python 环境

使用 Anaconda 管理 Python 环境：

```bash
# 激活 conda 环境
conda activate hystooth_flask
```

> **注意**：确保环境名称与实际环境一致，可通过 `conda env list` 查看所有环境。

### 3.3 步骤三：启动 uWSGI

uWSGI 作为 WSGI 服务器运行 Flask 应用：

```bash
# 进入项目目录
cd /home/rootroot/hystooth_flask

# 启动 uWSGI
uwsgi --ini uwsgi.ini
```

**uWSGI 配置说明** (`uwsgi.ini`)：

| 配置项 | 值 | 说明 |
|--------|-----|------|
| http | 127.0.0.1:8001 | HTTP 服务地址 |
| processes | 2 | 工作进程数 |
| threads | 8 | 每个进程的线程数 |
| buffer-size | 32768 | 缓冲区大小 |
| master | true | 启用主进程 |
| daemonize | uwsgi.log | 后台运行并记录日志 |

**验证 uWSGI 启动成功**：

```bash
# 检查 uWSGI 进程
ps aux | grep uwsgi

# 检查端口监听
netstat -tlnp | grep 8001
```

### 3.4 步骤四：启动 Celery

Celery 作为异步任务处理器，用于批量图片检测：

```bash
# 使用服务脚本启动 Celery
sh /etc/init.d/celeryd
```

> **注意**：确保 Celery 配置文件 `/etc/init.d/celeryd` 存在且可执行。

**Celery 配置说明** (在 `server.py` 中)：

```python
app.config['CELERY_BROKER_URL'] = 'redis://@127.0.0.1:6379/0'
app.config['CELERY_RESULT_BACKEND'] = 'redis://@127.0.0.1:6379/1'
```

### 3.5 步骤五：（可选）启动 Nginx

Nginx 作为反向代理，提供 Web 服务：

```bash
# 进入 Nginx 安装目录
cd /usr/local/nginx/sbin

# 启动 Nginx
./nginx
```

**Nginx 配置说明**：

- 监听端口：80
- 反向代理到：`http://127.0.0.1:8001`

## 5. 服务管理

### 5.1 启动服务

```bash
# 1. 启动 Redis
redis-server /etc/redis.conf

# 2. 激活 conda 环境
conda activate hystooth_flask

# 3. 启动 uWSGI
cd /home/rootroot/hystooth_flask
uwsgi --ini uwsgi.ini

# 4. 启动 Celery
sh /etc/init.d/celeryd

# 5. 启动 Nginx (可选)
cd /usr/local/nginx/sbin
./nginx
```

### 5.2 停止服务

```bash
# 停止 uWSGI
pkill -f uwsgi

# 停止 Redis
redis-cli shutdown

# 停止 Celery
pkill -f celery

# 停止 Nginx
cd /usr/local/nginx/sbin
./nginx -s stop
```

### 5.3 重启服务

```bash
# 重启 uWSGI
pkill -f uwsgi
cd /home/rootroot/hystooth_flask
uwsgi --ini uwsgi.ini

# 重启 Celery
pkill -f celery
sh /etc/init.d/celeryd

# 重启 Nginx
cd /usr/local/nginx/sbin
./nginx -s reload
```

## 6. 常见问题

### 6.1 端口占用

```bash
# 查看端口占用
netstat -tlnp | grep -E "80|6379|8001"

# 杀死占用进程
kill -9 <PID>
```

### 6.2 Redis 连接失败

```bash
# 检查 Redis 是否运行
redis-cli ping

# 启动 Redis
redis-server /etc/redis.conf
```

### 6.3 uWSGI 启动失败

```bash
# 检查日志
cat /home/rootroot/hystooth_flask/uwsgi.log

# 常见错误：
# - 端口被占用 → 修改 uwsgi.ini 中的端口
# - 模块导入错误 → 检查 Python 环境和依赖
```

### 6.4 Celery 任务未执行

```bash
# 检查 Celery worker 状态
celery -A server inspect active

# 查看 Celery 日志
tail -f /var/log/celery.log
```

## 7. 快速部署脚本

可创建一键部署脚本：

```bash
#!/bin/bash

echo "=== 牙齿疾病检测系统部署 ==="

# 启动 Redis
echo "[1/5] 启动 Redis..."
redis-server /etc/redis.conf &

# 激活环境并启动 uWSGI
echo "[2/5] 启动 uWSGI..."
cd /home/rootroot/hystooth_flask
conda activate hystooth_flask
uwsgi --ini uwsgi.ini &

# 启动 Celery
echo "[3/5] 启动 Celery..."
sh /etc/init.d/celeryd &

# 启动 Nginx
echo "[4/5] 启动 Nginx..."
cd /usr/local/nginx/sbin
./nginx

echo "[5/5] 检查服务状态..."
sleep 2
ps aux | grep -E "redis-server|uwsgi|celery|nginx" | grep -v grep

echo "=== 部署完成 ==="
```

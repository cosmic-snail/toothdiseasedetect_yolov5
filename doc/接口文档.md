# 牙齿疾病检测系统 - 接口文档

## 1. 接口概述

本文档描述了牙齿疾病检测系统提供的所有API接口，包括同步检测接口、异步检测接口的使用方法、请求格式、响应格式以及错误码说明。

### 1.1 基础信息

| 项目 | 说明 |
|------|------|
| 基础URL | `http://{server}:8001` |
| 协议 | HTTP |
| 编码 | UTF-8 |
| 请求方法 | POST |
| Content-Type | application/json |

---

## 2. 接口列表

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 同步报告接口 | POST | `/api/reports` | 同步处理牙齿图片检测 |
| 异步报告接口 | POST | `/api/asyncReports` | 异步处理批量牙齿图片检测 |

---

## 3. 同步报告接口

### 3.1 接口说明

- **接口路径**：`/api/reports`
- **请求方法**：`POST`
- **功能说明**：同步处理用户提交的牙齿图片，返回检测结果
- **适用场景**：单张或少量图片的实时检测需求

### 3.2 请求参数

#### 请求头

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| Content-Type | String | 是 | 固定值：`application/json` |

#### 请求体

```json
{
    "reportId": "报告ID",
    "images": [
        {
            "id": "图片唯一标识",
            "url": "图片URL地址"
        }
    ]
}
```

#### 参数说明

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| reportId | String | 否 | 报告唯一标识，用于关联检测结果 |
| images | Array | 是 | 图片列表，最多支持20张 |
| images[].id | String | 是 | 图片唯一标识，不能为空 |
| images[].url | String | 是 | 图片的完整URL地址，必须可访问 |

### 3.3 响应参数

#### 成功响应

```json
{
    "code": 0,
    "message": "解析完成",
    "data": {
        "reportId": "报告ID",
        "images": [
            {
                "id": "图片ID",
                "results": "4",
                "diseases": [
                    {
                        "code": "qc",
                        "name": "龋齿",
                        "result": "4",
                        "detail": {
                            "label": "qc",
                            "confidences": "0.85",
                            "minx:miny": [120, 80],
                            "maxx:maxy": [350, 280]
                        }
                    }
                ]
            }
        ]
    }
}
```

#### 响应字段说明

| 字段名 | 类型 | 说明 |
|--------|------|------|
| code | Integer | 状态码，0表示成功 |
| message | String | 状态消息 |
| data | Object | 响应数据体 |
| data.reportId | String | 报告ID |
| data.images | Array | 图片检测结果列表 |
| data.images[].id | String | 图片ID |
| data.images[].results | Integer/String | 检测结果代码 |
| data.images[].diseases | Array | 检测到的疾病列表 |
| data.images[].diseases[].code | String | 疾病代码 |
| data.images[].diseases[].name | String | 疾病名称 |
| data.images[].diseases[].result | Integer | 结果代码 |
| data.images[].diseases[].detail | Object | 详细信息 |
| data.images[].diseases[].detail.label | String | 标签名称 |
| data.images[].diseases[].detail.confidences | String | 置信度(0-1) |
| data.images[].diseases[].detail.minx:miny | Array | 边界框左上角坐标 [x, y] |
| data.images[].diseases[].detail.maxx:maxy | Array | 边界框右下角坐标 [x, y] |

#### 检测结果代码说明

| results值 | 含义 | 描述 |
|-----------|------|------|
| 1 | 检测到牙齿 | MobileNetV3分类结果，表示图片中有牙齿 |
| 2 | 未检测到牙齿 | 图片中未检测到牙齿区域 |
| 3 | 检测失败 | 解析异常或处理失败 |
| 4 | 检测到疾病 | YOLOv5检测到具体疾病 |

### 3.4 错误响应

```json
{
    "code": 2,
    "message": "url和id不能为空!"
}
```

### 3.5 请求示例

#### cURL示例

```bash
curl -X POST 'http://127.0.0.1:8001/api/reports' \
  -H 'Content-Type: application/json' \
  -d '{
    "reportId": "RPT20230901001",
    "images": [
        {
            "id": "IMG001",
            "url": "https://example.com/tooth1.jpg"
        },
        {
            "id": "IMG002",
            "url": "https://example.com/tooth2.jpg"
        }
    ]
}'
```

#### Python示例

```python
import requests
import json

url = "http://127.0.0.1:8001/api/reports"
headers = {"Content-Type": "application/json"}
payload = {
    "reportId": "RPT20230901001",
    "images": [
        {
            "id": "IMG001",
            "url": "https://example.com/tooth1.jpg"
        }
    ]
}

response = requests.post(url, headers=headers, json=payload)
print(response.json())
```

## 4. 异步报告接口

### 4.1 接口说明

- **接口路径**：`/api/asyncReports`
- **请求方法**：`POST`
- **功能说明**：异步处理批量牙齿图片检测请求，立即返回任务接收状态，实际检测在后台异步完成
- **适用场景**：大量图片的批量处理，不要求实时返回结果

### 4.2 请求参数

#### 请求头

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| Content-Type | String | 是 | 固定值：`application/json` |

#### 请求体

请求体格式与同步接口完全相同：

```json
{
    "reportId": "报告ID",
    "images": [
        {
            "id": "图片唯一标识",
            "url": "图片URL地址"
        }
    ]
}
```

### 4.3 响应参数

#### 成功响应

```json
{
    "code": 0,
    "message": "接收成功！"
}
```

异步接口仅返回任务接收状态，实际检测结果通过回调接口返回。

### 4.4 回调机制

异步任务完成后，系统会自动将检测结果回调到指定接口：

- **回调地址**：`https://app.yueyayun.com/apptoothbrush/ai/callback`
- **回调方法**：POST
- **回调内容类型**：application/json

#### 回调请求体

```json
{
    "reportId": "报告ID",
    "images": [
        {
            "id": "图片ID",
            "results": "4",
            "diseases": [...]
        }
    ]
}
```

### 4.5 错误响应

| 错误码 | 错误信息 | 说明 |
|--------|----------|------|
| 0 | 接收成功！ | 任务已接收 |
| 1 | 请求参数未传！ | 请求体为空 |
| 2 | url和id不能为空！ | 参数校验失败 |
| 3 | 接口服务出错，接收失败！ | 服务器内部错误 |

### 4.6 请求示例

#### cURL示例

```bash
curl -X POST 'http://127.0.0.1:8001/api/asyncReports' \
  -H 'Content-Type: application/json' \
  -d '{
    "reportId": "RPT20230901002",
    "images": [
        {
            "id": "IMG001",
            "url": "https://example.com/tooth1.jpg"
        },
        {
            "id": "IMG002",
            "url": "https://example.com/tooth2.jpg"
        },
        {
            "id": "IMG003",
            "url": "https://example.com/tooth3.jpg"
        }
    ]
}'
```

## 5. 错误码汇总

### 5.1 通用错误码

| 错误码 | 错误信息 | 说明 | 处理建议 |
|--------|----------|------|----------|
| 0 | 解析完成/接收成功 | 成功 | 正常处理响应数据 |
| 1 | 请求参数未传！ | 请求体为空 | 检查请求体是否正确传递 |
| 2 | url和id不能为空！ | 参数校验失败 | 检查images数组中每个元素是否包含id和url |
| 3 | 接口服务出错，接收失败！ | 服务器内部错误 | 稍后重试或联系管理员 |

### 5.2 检测结果码

| results值 | 含义 | 可能的疾病类型 |
|-----------|------|----------------|
| 1 | 检测到牙齿 | 存在牙齿区域 |
| 2 | 未检测到牙齿 | 可能是非牙齿图片或拍摄不规范 |
| 3 | 检测失败 | 图片质量问题或网络异常 |
| 4 | 检测到疾病 | 见下表 |

### 5.3 疾病代码对照表

| 疾病代码 | 疾病名称 | 描述 |
|----------|----------|------|
| tooth | 牙齿 | 基础类别 |
| qc | 龋齿 | 蛀牙、龋洞 |
| yjs | 牙结石 | 牙石、牙垢 |
| ssy | 色素牙 | 牙齿着色 |
| zc | 智齿 | 智齿问题 |
| xzqs | 楔状缺损 | 牙齿楔形缺损 |
| ytqs | 牙体缺损 | 牙齿部分缺失 |
| ycms | 牙齿磨损 | 牙齿磨耗 |
| yyl | 牙隐裂 | 牙齿裂纹 |
| yyy | 牙龈炎 | 牙龈炎症 |
| kqky | 口腔溃疡 | 口腔黏膜溃疡 |
| yzws | 牙周萎缩 | 牙龈萎缩 |
| ylbq | 牙列不齐 | 牙齿排列不整齐 |
| ylxs | 牙列稀疏 | 牙齿间隙大 |
| spy | 双排牙 | 乳牙滞留 |
| ycqs | 牙齿缺失 | 缺牙 |

## 6. 使用约束

### 6.1 接口限制

| 限制项 | 限制值 | 说明 |
|--------|--------|------|
| 单次请求最大图片数 | 20张 | 同步接口建议值 |
| 图片URL长度 | 不超过2048字符 | HTTP协议限制 |
| 图片文件大小 | 建议不超过10MB | 过大图片影响处理速度 |
| 请求超时 | 180秒 | Gunicorn配置 |

### 6.2 图片输入规格

#### 6.2.1 支持的图片格式

| 格式 | 扩展名 | 支持状态 |
|------|--------|----------|
| JPEG | jpg, jpeg | ✅ 支持 |
| PNG | png | ✅ 支持 |
| BMP | bmp | ✅ 支持 |

> **注意**：系统通过文件扩展名进行格式校验，请确保URL指向的文件扩展名正确。

#### 6.2.2 图片尺寸规格

| 规格项 | 要求 | 备注 |
|--------|------|------|
| 推荐最小分辨率 | 640 × 640 像素 | 低于此分辨率可能影响检测准确性 |
| 推荐最佳分辨率 | 1280 × 1280 像素 | 清晰度与处理速度的最佳平衡点 |
| 宽高比 | 无限制 | 系统会自动进行尺寸适配 |

#### 6.2.3 图片文件大小

| 规格项 | 要求 | 备注 |
|--------|------|------|
| 推荐最大文件大小 | 10 MB | 超过可能导致下载超时 |
| 最小文件大小 | 1 KB | 过小的文件可能已损坏 |

#### 6.2.4 图片URL要求

| 要求项 | 说明 |
|--------|------|
| 协议 | 必须为 `http://` 或 `https://` 开头 |
| 可访问性 | 必须为公开可访问的URL（无需认证） |
| 响应状态 | 图片URL必须返回 HTTP 200 状态码 |
| 响应时间 | 建议10秒内返回图片数据 |

#### 6.2.5 图片内容拍摄要求

为了获得最佳的检测效果，请按以下要求拍摄牙齿图片：

| 要求 | 说明 | 不符合示例 |
|------|------|------------|
| 拍摄主体 | 牙齿/口腔区域占据图片主要部分 | 脸部照片、风景照 |
| 清晰度 | 牙齿区域清晰可见，无模糊 | 失焦、运动模糊 |
| 光线 | 充足且均匀照明 | 过度暗光、背光、强光反射 |
| 角度 | 正对口腔拍摄 | 侧面拍摄、倾斜角度 |
| 遮挡 | 牙齿区域无大面积遮挡 | 嘴唇、舌头遮挡牙齿 |
| 背景 | 简洁的背景 | 过于复杂的背景干扰识别 |
| 张口程度 | 需张开口腔露出牙齿 | 闭口照片 |

**拍摄建议**：
- 建议在自然光或均匀人工光源下拍摄
- 拍摄时保持手机/相机稳定
- 可拍摄多张不同角度的照片提高检测准确性
- 如有假牙、牙套等，请确保在图片中可见以便模型识别

### 6.3 注意事项

1. **同步接口**：适用于实时检测场景，但大量图片时可能导致请求超时
2. **异步接口**：适用于批量处理，检测结果通过回调返回，需要确保回调地址可访问
3. **图片URL**：必须是可公开访问的URL，系统会通过URL下载图片进行处理
4. **临时文件**：系统会在处理完成后自动清理临时文件，不占用服务器存储

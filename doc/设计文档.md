# 牙齿疾病检测系统 - 设计文档

## 1. 项目概述

### 1.1 项目背景

牙齿疾病检测系统是一个基于深度学习的口腔健康检测服务，旨在自动识别和分析用户拍摄的牙齿图片，检测多种常见的牙齿疾病和口腔问题，为用户提供快速、便捷的口腔健康评估。

### 1.2 项目目标

- 提供高效的牙齿疾病自动检测能力
- 支持同步和异步两种检测模式
- 集成多种深度学习模型，提高检测准确率
- 提供标准化的RESTful API接口

## 3. 功能设计

### 3.1 核心功能

#### 3.1.1 牙齿疾病检测

系统支持检测以下16种牙齿疾病和口腔问题：

| 疾病代码 | 疾病名称 | 说明 |
|----------|----------|------|
| tooth | 牙齿 | 基础类别 |
| qc | 龋齿 | 蛀牙、龋洞 |
| yjs | 牙结石 | 牙石、牙垢 |
| ssy | 色素牙 | 牙齿着色 |
| zc | 智齿 | 智齿问题 |
| xzqs | 楔状缺损 | 牙齿楔形缺损 |
| ytqs | 牙体缺损 | 牙齿部分缺失 |
| ycms | 牙齿磨损 | 牙齿磨耗 |
| yyl | 牙隐裂 | 牙齿裂纹 |
| yyy | 牙龈炎 | 牙龈炎症 |
| kqky | 口腔溃疡 | 口腔黏膜溃疡 |
| yzws | 牙周萎缩 | 牙龈萎缩 |
| ylbq | 牙列不齐 | 牙齿排列不整齐 |
| ylxs | 牙列稀疏 | 牙齿间隙大 |
| spy | 双排牙 | 乳牙滞留 |
| ycqs | 牙齿缺失 | 缺牙 |

#### 3.1.2 检测流程

```
输入图片
    │
    ▼
┌─────────────────┐
│ MobileNetV3    │ ──► 判断是否有牙齿 (阈值: 0.75)
│ (牙齿初筛模型)  │
└─────────────────┘
    │
    ▼
┌─────────────────┐
│ YOLOv5          │ ──► 检测具体疾病类型
│ (疾病检测模型)  │
└─────────────────┘
    │
    ▼
┌─────────────────┐
│ 亮度自适应重试  │ ──► 如未检测到疾病，调整亮度重试
│ (多层级亮度)   │
└─────────────────┘
    │
    ▼
检测结果输出
```

#### 3.1.3 亮度自适应重试机制

当YOLOv5未能检测出疾病时，系统会自动调整图像亮度进行重试：

- 亮度调整层级：`[-40, -20, 0, 20, 40, 60, 80, 100]`
- 饱和度固定：`100`
- 重试策略：逐级调整亮度，直到检测出疾病或遍历所有层级

### 3.2 接口设计

#### 3.2.1 同步报告接口 (`/api/reports`)

- **功能**：同步处理牙齿图片检测请求
- **适用场景**：单张或少量图片的实时检测

#### 3.2.2 异步报告接口 (`/api/asyncReports`)

- **功能**：异步处理批量牙齿图片检测请求
- **适用场景**：大量图片的批量处理
- **回调机制**：检测完成后自动回调指定接口

## 6. 边界约束与限制

### 6.1 输入约束

#### 6.1.1 图片格式规格

| 约束项 | 限制说明 | 备注 |
|--------|----------|------|
| 支持的图片格式 | png, jpg, JPG, PNG, bmp | 区分大小写 |
| 推荐格式 | JPG, PNG | 兼容性最好 |
| 文件扩展名校验 | 按扩展名白名单过滤 | 不校验文件内容 |

#### 6.1.2 图片尺寸规格

| 约束项 | 限制说明 | 备注 |
|--------|----------|------|
| 推荐最小分辨率 | 640 x 640 像素 | 低于此分辨率可能影响检测准确性 |
| 推荐最佳分辨率 | 1280 x 1280 像素 | 清晰度与处理速度的平衡点 |
| 最大分辨率 | 无硬性限制 | 过大的图片会延长处理时间 |
| 宽高比 | 无限制 | 系统会自动进行尺寸适配 |

#### 6.1.3 图片文件大小

| 约束项 | 限制说明 | 备注 |
|--------|----------|------|
| 推荐最大文件大小 | 10 MB | 超过可能影响下载和处理速度 |
| 最小文件大小 | 1 KB | 过小的文件可能是损坏的图片 |
| 实际限制 | 无硬性限制 | 取决于网络传输和服务器资源 |

#### 6.1.4 图片内容要求

| 要求项 | 说明 | 注意事项 |
|--------|------|----------|
| 拍摄主体 | 牙齿/口腔区域 | 需清晰展示牙齿 |
| 焦点 | 牙齿区域清晰 | 避免模糊、失焦 |
| 光线 | 充足且均匀 | 避免过度暗光或强光 |
| 角度 | 正对口腔 | 侧面或遮挡会影响检测 |
| 背景 | 简洁 | 背景过于复杂可能干扰识别 |
| 遮挡 | 无大面积遮挡 | 嘴唇、舌头等不应遮挡牙齿 |

#### 6.1.5 URL规格

| 约束项 | 限制说明 | 备注 |
|--------|----------|------|
| 协议 | HTTP, HTTPS | 必须以 http:// 或 https:// 开头 |
| URL长度 | 最大 2048 字符 | 符合 RFC 3986 |
| 访问要求 | 公开可访问 | 不需要认证/鉴权 |
| 响应时间 | 建议 10 秒内 | 超时可能导致下载失败 |
| 状态码 | 200 OK | 其他状态码视为下载失败 |

#### 6.1.6 其他输入约束

| 约束项 | 限制说明 |
|--------|----------|
| 图片ID | 必须为非空字符串，建议使用英文、数字、下划线 |
| 单次请求图片数量 | 建议不超过20张（同步接口），异步接口无严格限制 |
| reportId | 可选，建议唯一标识每次请求 |

### 6.2 服务约束

| 约束项 | 限制说明 |
|--------|----------|
| 同步接口超时 | 建议单张图片处理不超过30秒 |
| 异步接口响应 | 立即返回任务接收状态，实际处理异步完成 |
| 模型推理设备 | 自动检测CUDA，可用则使用GPU，否则使用CPU |
| 临时文件 | 使用临时目录存储，函数结束后自动清理 |

### 6.3 错误处理

| 错误码 | 错误信息 | 说明 |
|--------|----------|------|
| 0 | 解析成功 | 正常响应 |
| 1 | 请求参数未传 | 请求体为空 |
| 2 | url和id不能为空 | 参数校验失败 |
| 3 | 接口服务出错 | 服务器内部错误 |

### 6.4 亮度重试约束

- 最大重试次数：8次（对应8个亮度层级）
- 亮度调整范围：-40 到 100，步长20
- 饱和度调整：固定100%

## 7. 模型说明

### 7.1 YOLOv5模型

- **模型文件**：`weights/last.pt`
- **配置文件**：`data/qc.yaml`
- **检测类别**：4类（qc, yjs, ylbq, ssy）
- **输入尺寸**：640x640
- **置信度阈值**：0.25
- **NMS IOU阈值**：0.45

### 7.2 MobileNetV3模型

- **模型文件**：`mobilenetv3/runs/MobileNetV3_tooth.pth`
- **分类类别**：2类（none_tooth, tooth）
- **输入尺寸**：224x224
- **牙齿判断阈值**：0.75

## 8. 日志设计

### 8.1 日志级别

- DEBUG：详细调试信息
- INFO：常规操作信息
- WARNING：警告信息（如检测失败但可重试）
- ERROR：错误信息（如致命异常）

### 8.2 日志格式

```
时间戳 文件名 [行号] 函数名 [日志级别] 消息内容
```

### 8.3 关键日志记录点

- 请求参数校验
- 图片下载状态
- 模型推理结果
- 异常捕获详情
- 异步任务回调结果
